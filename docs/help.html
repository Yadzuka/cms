Здесь описан весь процесс конфигерирования cms проекта пошагово. 1 - ый шаг Загрузить сторонние библиотеки через make contrib-lib в корневом каталоге. 2 - ой шаг Собрать пакеты проекта, перед этим создав папку lib в webapps/cms/WEB-INF, - make jar в корневом каталоге 3 - ий шаг Настроить web.xml.sample и сделать от него копию web.xml. Параметры подлежащие изменению: - Display name - отображаемое имя - QRDB_PATH - параметр для выбора вашей файловой системы (где будут храниться все данные) - role name - роль, которая должна быть добавлена в tomcat-users.xml - realm name - также имя, которое выводится в авторизации - security role -> role name - то же имя роли в tomcat-users.xml - root и user - первое - корневой каталог файловой системы (такой же как и qrdb_path), второе - имя папки для пользователя (обычная папка или несколько папок) - logFilePath - файл для записи логов (загрузки, ошибок, сохранения) - файл необходимо преждевременно создать, указать к нему полный путь и не забыть дать права на запись в этот файл серверу приложений 4 - ый шаг - добавление симлинка на папку в веб приложения сервера приложений или копирование папки в место, куда смотрит веб сервер
